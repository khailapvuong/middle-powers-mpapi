# M-PAPI Power BI dashboard — build tutorial

Step-by-step instructions for assembling the 2-page MVP in Power BI Desktop using the data layer at `outputs/pbi/`. Expected build time: **~3 hours** for a builder comfortable with PBI Desktop; ~5 hours for a first-time PBI user.

Companion document: `docs/PBI_DASHBOARD_SPEC.md` (the visual + measure spec). Refer to that document for the *what*; this document covers the *how*.

## Prerequisites

| Requirement | Version | Notes |
|---|---|---|
| Power BI Desktop | 2024.06 or later | Free download from `https://powerbi.microsoft.com/desktop/`. Pro licence not needed to build; only needed to publish to PBI Service. |
| Repo checkout | Latest `main` | `git clone https://github.com/khailapvuong/middle-powers-mpapi.git` |
| Data layer | Generated by §20 of `M-PAPI.ipynb` | Run `jupyter execute --inplace M-PAPI.ipynb` once after cloning; populates `outputs/pbi/` |
| Display | 1920×1080+ recommended | The 2-page layout assumes a 16:9 canvas |

Verify the data layer is in place before starting:

```bash
ls outputs/pbi/
# Should show: dim_country.csv, dim_indicator.csv, dim_action.csv,
#              fact_index_long.csv, fact_vulnerability_long.csv, fact_shap_long.csv,
#              fact_counterfactual.csv, fact_sensitivity.csv, README.md
```

## Step 1 — Open Power BI Desktop and load the data layer

1. Launch Power BI Desktop.
2. On the splash screen, click **Get data** → **More** → **Folder** → **Connect**.
3. Browse to the repo's `outputs/pbi/` folder. Click **OK**.
4. In the preview window, click **Combine & Load** at the bottom right.

PBI auto-detects each CSV as a separate table. After load, the **Data** pane on the right should list 8 tables:

- `dim_country`
- `dim_indicator`
- `dim_action`
- `fact_index_long`
- `fact_vulnerability_long`
- `fact_shap_long`
- `fact_counterfactual`
- `fact_sensitivity`

Plus a few auto-generated helper queries from the folder import (`Sample File`, `Transform File`, etc.) — these can be hidden but not deleted.

**Verify column types.** Click each fact table in the Data pane → verify numeric columns (`score`, `rank`, `shapley_value`, `delta_composite`) loaded as Decimal or Whole Number. If any loaded as Text, right-click the column → Change Type → Decimal Number.

## Step 2 — Set up the relationships

Switch to **Model view** (bottom-left icon, three connected boxes).

Drag the following relationships in this exact order. Each is one-to-many (`dim` → `fact`), single-direction:

| From (one) | To (many) | Cardinality | Filter direction |
|---|---|---|---|
| `dim_country[iso3]` | `fact_index_long[iso3]` | 1 to many | Single (dim → fact) |
| `dim_country[iso3]` | `fact_vulnerability_long[iso3]` | 1 to many | Single |
| `dim_country[iso3]` | `fact_shap_long[iso3]` | 1 to many | Single |
| `dim_country[iso3]` | `fact_counterfactual[iso3]` | 1 to many | Single |
| `dim_country[iso3]` | `fact_sensitivity[iso3]` | 1 to many | Single |
| `dim_indicator[indicator_id]` | `fact_shap_long[indicator_id]` | 1 to many | Single |
| `dim_action[action_key]` | `fact_counterfactual[action_key]` | 1 to many | Single |

Double-click each relationship line to verify "Cross filter direction" is set to **Single**, not Both. Bi-directional filters cause ambiguity warnings in DAX measures and are not needed for this model.

**Hide foreign-key columns from the report view** so users don't see raw ISO codes in slicers:

- `fact_index_long[iso3]` → right-click → Hide in report view
- `fact_vulnerability_long[iso3]` → Hide
- `fact_shap_long[iso3]`, `fact_shap_long[indicator_id]` → Hide
- `fact_counterfactual[iso3]`, `fact_counterfactual[action_key]` → Hide
- `fact_sensitivity[iso3]` → Hide

## Step 3 — Build Page 1 (Overview)

Switch to **Report view** (top-left icon). The default page is named "Page 1" — right-click the tab → Rename → **Overview**.

### 3.1 Create the DAX measures

In the Data pane, right-click `fact_index_long` → New measure. Paste each measure from `PBI_DASHBOARD_SPEC.md` § "Page 1 DAX measures":

```dax
Composite Score =
    CALCULATE(
        SUM(fact_index_long[score]),
        fact_index_long[axis_code] = "composite"
    )
```

Repeat for `[Capacity Z]`, `[Governance Z]`, `[Infrastructure Z]`, `[Rank]`. Each new measure attaches to `fact_index_long` (visible in the Data pane under that table).

For `[Top Country]`, `[Bottom Country]`, `[Top-5 Set]`, attach to the same `fact_index_long` table.

### 3.2 Title text box

Insert → Text box → place at top of page. Two lines:

- Line 1, 16pt bold: `M-PAPI: Middle-Power AI Proliferation Preparedness Index`
- Line 2, 11pt regular: `Companion to Teague, Ali, Sfeir, Fort (2026) — github.com/khailapvuong/middle-powers-mpapi`

### 3.3 KPI cards (3 cards)

For each card:

1. Visualizations pane → **Card** (not "Multi-row card").
2. Drag the measure (`[Top Country]`, `[Bottom Country]`, `[Top-5 Set]`) to Fields.
3. Format pane → Title → switch ON → enter "Rank-1 country" / "Rank-14 country" / "Top-5 set".

Stack the 3 cards vertically on the upper-left third of the canvas.

### 3.4 Weighting-scheme slicer

1. Visualizations pane → **Slicer**.
2. Drag `fact_index_long[scheme_label]` to Field.
3. Format pane → Slicer style → **Horizontal**.
4. Format pane → Selection → Single select = **On**.
5. In the slicer UI itself, click "Literature-elicited" to set the default.

Place the slicer in the upper-right third.

### 3.5 Sortable ranking table

1. Visualizations pane → **Table** (not "Matrix").
2. Drag in this order into Columns:
   - `dim_country[iso3]`
   - `dim_country[name]`
   - `[Capacity Z]`
   - `[Governance Z]`
   - `[Infrastructure Z]`
   - `[Composite Score]`
   - `[Rank]`
3. Click the column header of `[Rank]` to sort ascending.
4. Apply **conditional formatting** to `[Capacity Z]`, `[Governance Z]`, `[Infrastructure Z]`: Format pane → Cell elements → Background colour → On. Choose a diverging palette (red 0 → yellow → blue) with midpoint 0.
5. Place the table on the centre-left of the canvas, ~1/3 of canvas width.

### 3.6 Interactive typology scatter

The spec calls for marker shape variation by `is_aisi_network`. PBI's native scatter does not support shape-by-category, so we use the **two-layer workaround**:

**Layer A — AISI members (circles):**

1. Visualizations pane → **Scatter chart**.
2. Drag `dim_country[name]` to Values.
3. Drag `[Capacity Z]` to X-axis.
4. Drag `[Infrastructure Z]` to Y-axis.
5. Drag `dim_country[archetype]` to Legend.
6. Filters pane → Filters on this visual → Drag `dim_country[is_aisi_network]` → set filter to `is 1`.
7. Format pane → Markers → Shape → **Circle**, Size → 12.

**Layer B — non-AISI (diamonds):**

1. Copy-paste Layer A's visual (Ctrl+C, Ctrl+V) — preserves all field bindings.
2. Filters → change `is_aisi_network` filter to `is 0`.
3. Format pane → Markers → Shape → **Diamond**.
4. Format pane → Background → **Transparent** (so it stacks on top of Layer A).

Align both visuals to exactly the same canvas region.

Add a separate text legend below the chart (text box): "● circle = AISI Network member · ◆ diamond = not in Network".

Place this combined visual on the centre-right of the canvas.

### 3.7 Robustness CI bar chart (bottom strip)

Source: load `outputs/robustness_summary_with_ci.csv` directly as a 9th table (Get Data → Text/CSV).

1. Visualizations pane → **Clustered bar chart**.
2. Drag `robustness_summary_with_ci[perturbation]` to Y-axis.
3. Drag `robustness_summary_with_ci[rho]` to X-axis.
4. Drag `robustness_summary_with_ci[ci_lower_95]` and `robustness_summary_with_ci[ci_upper_95]` to X-axis as additional values — they render as adjacent bars.
5. Format → Reference lines → Add line at `robustness_summary_with_ci[threshold]` per row (PBI 2024+ supports dynamic reference lines from measure).
6. Title: "Robustness — Spearman ρ with Fisher-z 95% CI · all four CI lower bounds clear thresholds"

Place at the bottom of the canvas, full width.

## Step 4 — Build Page 2 (Country drill-through)

Insert → New page → rename to **Country drill-through**.

### 4.1 Country selector slicer

1. Visualizations pane → **Slicer**.
2. Drag `dim_country[name]` to Field.
3. Format pane → Slicer style → **Dropdown**.
4. Format pane → Selection → Single select = **On**, Show Select all = **Off**.
5. Place at the top-centre of the page. Default: "South Korea" (click it once to pre-select).

### 4.2 Per-axis horizontal bar (V2.2)

Create measure `[Axis Score]`:

```dax
Axis Score =
    VAR _selected_axis = SELECTEDVALUE(fact_index_long[axis_code])
    RETURN
        CALCULATE(
            SUM(fact_index_long[score]),
            fact_index_long[is_composite] = 0,
            fact_index_long[axis_code] = _selected_axis
        )
```

1. Visualizations pane → **Clustered bar chart**.
2. Drag `fact_index_long[axis_label]` to Y-axis.
3. Drag `[Axis Score]` to X-axis.
4. Filters on this visual → `fact_index_long[is_composite]` = 0.
5. Conditional formatting → Bars → diverging palette (red < 0, blue > 0).
6. Title: dynamic — Format → Title → set to text `"Axis profile for "` and concatenate with `SELECTEDVALUE(dim_country[name])` via a Calculated text field, or simply hard-code "Axis profile (selected country)".

Place on middle-left, ~40% of canvas width.

### 4.3 Per-vector vertical bar (V2.3)

Create measures:

```dax
Vector Score = SUM(fact_vulnerability_long[score])
Vector Rank = SUM(fact_vulnerability_long[rank])
```

1. Visualizations pane → **Clustered column chart**.
2. Drag `fact_vulnerability_long[vector_label]` to X-axis.
3. Drag `[Vector Score]` to Y-axis.
4. Data labels → On → show value + rank suffix: format string `"#0.00 (rank #")"` (PBI custom format) — or add a separate text label.
5. Title: "Vulnerability profile (selected country)".

Place on middle-right.

### 4.4 Shapley waterfall (V2.4)

Create measure:

```dax
Shapley Value = SUM(fact_shap_long[shapley_value])
```

1. Visualizations pane → **Waterfall chart**.
2. Drag `dim_indicator[indicator_id]` to Category.
3. Drag `[Shapley Value]` to Y-axis.
4. Sort: descending by absolute value (click the … menu → Sort by → `[Shapley Value]` descending).
5. Format → Sentiment colours → Increase = green, Decrease = red, Total = grey.
6. Title: "Shapley attribution (selected country)".

Place at bottom-left.

### 4.5 Counterfactual what-if (V2.5) — KILLER DEMO

**Step 4.5.1 — Create 5 what-if parameters**

Modelling tab → New parameter → Numeric range. Repeat 5 times:

| Parameter name | Min | Max | Increment | Default |
|---|---|---|---|---|
| `WhatIf_JoinAISI` | 0 | 1 | 1 | 0 |
| `WhatIf_PublishStrategy` | 0 | 1 | 1 | 0 |
| `WhatIf_JoinAustraliaGroup` | 0 | 1 | 1 | 0 |
| `WhatIf_SignLabMoU` | 0 | 1 | 1 | 0 |
| `WhatIf_FosterIGSCFirm` | 0 | 1 | 1 | 0 |

Tick "Add slicer to this page" for each. Five toggle slicers appear; arrange them stacked on the right side of the page.

**Step 4.5.2 — Create the delta + post-action composite measures**

```dax
Selected Action Delta =
    VAR _delta_aisi      = COALESCE(CALCULATE(SUM(fact_counterfactual[delta_composite]), fact_counterfactual[action_key] = "JOIN_AISI"), 0) * [WhatIf_JoinAISI Value]
    VAR _delta_strategy  = COALESCE(CALCULATE(SUM(fact_counterfactual[delta_composite]), fact_counterfactual[action_key] = "PUBLISH_AI_STRATEGY"), 0) * [WhatIf_PublishStrategy Value]
    VAR _delta_aus_grp   = COALESCE(CALCULATE(SUM(fact_counterfactual[delta_composite]), fact_counterfactual[action_key] = "JOIN_AUSTRALIA_GROUP"), 0) * [WhatIf_JoinAustraliaGroup Value]
    VAR _delta_mou       = COALESCE(CALCULATE(SUM(fact_counterfactual[delta_composite]), fact_counterfactual[action_key] = "SIGN_LAB_MOU"), 0) * [WhatIf_SignLabMoU Value]
    VAR _delta_igsc      = COALESCE(CALCULATE(SUM(fact_counterfactual[delta_composite]), fact_counterfactual[action_key] = "FOSTER_IGSC_FIRM"), 0) * [WhatIf_FosterIGSCFirm Value]
    RETURN _delta_aisi + _delta_strategy + _delta_aus_grp + _delta_mou + _delta_igsc

New Composite If Acted =
    CALCULATE([Composite Score], fact_index_long[scheme_code] = "literature") + [Selected Action Delta]
```

**Step 4.5.3 — Two KPI cards**

1. Insert two **Card** visuals side by side.
2. Card A — value = `[Composite Score]` (filter by `scheme_code = "literature"`); title "Composite (baseline)".
3. Card B — value = `[New Composite If Acted]`; title "Composite (post-action)".
4. Format Card B → Background → conditional formatting: green if `[New Composite If Acted] > [Composite Score]`, else default.

**Step 4.5.4 — Action descriptions table**

Add a small table visual listing all 5 actions for reference:

1. Visualizations pane → **Table**.
2. Drag `dim_action[action_key]`, `dim_action[description]`, `dim_action[target_axis]`, and a measure `[Action Delta] = SUM(fact_counterfactual[delta_composite])` filtered to the current country.
3. Title: "Per-action deltas (filtered to selected country)".

Place below the cards.

**Step 4.5.5 — Methodological caveat**

Insert → Text box. Body:

> The what-if combines single-action deltas linearly. Each `delta_composite` is a marginal-effect estimate (one action, all other indicators held constant). True multi-action composition is non-linear under geometric across-axis aggregation; for combined-action scenarios consult §13.4's per-action `delta_axis_score` (free of the §11 geometric-shift artifact) and re-run the notebook with simultaneous indicator updates.

### 4.6 MC rank IQR cards (V2.6)

Create 4 measures:

```dax
Rank Median = SUM(fact_sensitivity[rank_median])
Rank p10 = SUM(fact_sensitivity[rank_p10])
Rank p90 = SUM(fact_sensitivity[rank_p90])
Rank IQR = SUM(fact_sensitivity[rank_iqr])

Rank Interpretation =
    VAR _iqr = [Rank IQR]
    RETURN
        SWITCH(
            TRUE(),
            _iqr <= 1, "Rank is highly stable across 10,000 weight draws (IQR ≤ 1).",
            _iqr <= 2, "Rank is moderately stable; small re-ordering under perturbation.",
            _iqr <= 3, "Rank is weight-sensitive; cite as a range, not a point.",
            "Rank cycles substantially under perturbation; treat with caution."
        )
```

Place 3 KPI cards in a row (Median, p10, p90), then a text-only card below with `[Rank Interpretation]`.

Place at the bottom-full-width of the page.

Before declaring the build done, run these checks:

1. **Page 1 rank-1 KPI** reads `United Kingdom` when the slicer is set to "Literature-elicited". Switch to "PCA-derived" — still UK. Switch to "Equal" — still UK.
2. **Page 1 typology scatter** shows 14 markers. Five circles (AISI members: UK, Canada, France, Japan, Singapore, South Korea, EU — actually 7) in the upper-right quadrant; the remaining countries as diamonds.
3. **Page 2 with South Korea selected**: per-axis bar shows Capacity ≈ +0.94, Governance ≈ +0.60, Infrastructure ≈ +0.49. Per-vector bar shows Cyber, CBRN, Influence all positive.
4. **Page 2 what-if smoke test**: with Singapore selected and `WhatIf_JoinAustraliaGroup` toggled to 1, the "Composite (post-action)" card increases from ~1.81 to ~1.91 (matching `outputs/counterfactual_scenarios.csv` row SGP × JOIN_AUSTRALIA_GROUP).
5. **Refresh round-trip**: Home → Refresh. All values update without errors. Verify the rank-1 KPI still reads UK.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Numeric columns import as Text | Locale-dependent CSV parsing | Right-click column → Change Type → Decimal Number, or use Locale = "English (United States)" in Power Query |
| Scatter shows only one shape | Two-layer trick not applied | Re-check Step 3.6 — two scatter visuals, one filtered to `is_aisi_network = 1`, one to `= 0`, with the diamond layer having a transparent background and stacked over the circle layer |
| What-if parameter doesn't update card | Measure not referencing the parameter `[Value]` measure | Each parameter auto-creates `[<param-name> Value]` — make sure the `Selected Action Delta` references those, not the parameter table |
| Relationship ambiguity warning | A bi-directional filter accidentally enabled | Model view → edit relationship → set Cross filter direction = Single |
| Title bar shows "Power BI Desktop (Unsupported version)" | PBI Desktop is older than 2024.06 (what-if parameter UX differs in older versions) | Update to the latest Desktop release |

## Refresh workflow

When `outputs/pbi/*.csv` changes (after re-running the notebook):

1. Open the `.pbix`.
2. Home tab → Refresh.
3. Save.

If you publish to PBI Service, configure an on-premises gateway pointing at the repo checkout for scheduled refresh — out of scope for v1.
